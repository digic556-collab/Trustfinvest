DEPLOYMENT OPTIONS
═══════════════════════════════════════════════════════════════════════════════

OPTION 1: VERCEL (FASTEST - RECOMMENDED)
Time: 5-10 minutes
Cost: Free tier available
Setup: Nearly automatic

  1. Push to GitHub:
     git add .
     git commit -m "Ready for deployment"
     git push origin main

  2. Go to vercel.com
     Click "New Project"
     Select your GitHub repository
     Click "Import"

  3. Add environment variables:
     Settings → Environment Variables
     Paste all from .env file

  4. Deploy
     Vercel auto-deploys on push

  5. Test
     Check file access returns 404
     Verify security headers present

Result: Your site lives at: https://projectname.vercel.app


OPTION 2: CPANEL/TRADITIONAL HOSTING
Time: 15-30 minutes
Cost: Monthly hosting fee
Setup: Manual upload

  1. Get FTP credentials from hosting provider

  2. Upload files via FTP:
     Use FileZilla
     Connect to FTP server
     Upload to: public_html/
     Ensure .htaccess is uploaded

  3. Verify .htaccess
     Show hidden files (Ctrl+H)
     Confirm .htaccess in public_html/
     Check permissions: 644

  4. Set up SSL:
     cPanel → AutoSSL → Run AutoSSL
     Or use Let's Encrypt

  5. Test
     Visit yourdomain.com
     Check file access returns 404

Result: Your site lives at: https://yourdomain.com


OPTION 3: GITHUB PAGES
Time: 3-5 minutes
Cost: Free
Setup: Very simple

  1. Push code to GitHub
     git push origin main

  2. Go to repository settings

  3. Enable GitHub Pages:
     Source: main branch
     Folder: / (root)

  4. Done!
     Site available at: https://username.github.io/repo-name

Limitation: Static site only (no backend)


OPTION 4: DOCKER
Time: 20-40 minutes
Cost: Varies by host
Setup: Container deployment

  1. Build image:
     docker build -t trustfinvest-website .

  2. Test locally:
     docker run -d -p 80:80 trustfinvest-website

  3. Push to registry:
     docker push username/trustfinvest-website

  4. Deploy to host:
     docker run -d -p 80:80 username/trustfinvest-website

Result: Containerized deployment with full control

TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

Issue: Files return 403 instead of 404
Solution: .htaccess permissions wrong, contact hosting support

Issue: Website doesn't load
Solution: Check if index.html exists, verify RewriteEngine enabled

Issue: CSS/JS not loading (but not 404)
Solution: Check if .htaccess is actually deployed

Issue: Backend not working
Solution: Verify environment variables in hosting dashboard

Issue: Email not sending
Solution: Check SMTP credentials in .env, verify credentials in dashboard
