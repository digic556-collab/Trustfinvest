                        QUICK START OPTIONS
═══════════════════════════════════════════════════════════════════════════════

OPTION 1: RUN LOCALLY FOR TESTING (5 minutes)

  Step 1: Install dependencies
    npm install

  Step 2: Start backend server
    npm run dev
    (or: npm start)

  Step 3: Server running on
    http://localhost:5000

  Step 4: Check status
    curl http://localhost:5000/health
    curl http://localhost:5000/api/docs

  Step 5: View website
    - Static files served at port 5000
    - Open browser: http://localhost:5000
    - Or use: npx http-server .


OPTION 2: DEPLOY TO VERCEL (5-10 minutes) ⚡ FASTEST

  Step 1: Push to GitHub
    git add .
    git commit -m "Ready for production deployment"
    git push origin main

  Step 2: Go to vercel.com
    Click "New Project"
    Select GitHub repository
    Click "Import"

  Step 3: Add environment variables
    Go to Settings → Environment Variables
    Add all variables from .env file
    (FIREBASE_API_KEY, SMTP_HOST, SMTP_USER, SMTP_PASSWORD, etc.)

  Step 4: Deploy
    Click "Deploy"
    Wait 2-3 minutes
    Site is LIVE!

  Step 5: Test deployment
    curl -I https://yourdomain.vercel.app/css/style.css  (Should be 404)
    curl https://yourdomain.vercel.app/health             (Should show status)


OPTION 3: DEPLOY TO TRADITIONAL HOSTING (15-30 minutes)

  Step 1: Package files
    zip -r trustfinvest.zip *

  Step 2: Upload via FTP
    Get FTP credentials from hosting provider
    Use FileZilla or similar
    Upload to public_html/
    Ensure .htaccess is uploaded

  Step 3: Verify .htaccess
    In FTP: Show hidden files (Ctrl+H)
    .htaccess should be in public_html/
    Check permissions: 644

  Step 4: Set environment variables
    Create .env in project root (don't upload)
    Or use cPanel environment variable feature

  Step 5: Access your site
    Visit https://yourdomain.com
    Should load normally
