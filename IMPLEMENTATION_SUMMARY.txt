🚀 HOW IT WORKS (STEP-BY-STEP)
═══════════════════════════════════════════════════════════════════════════════

1. USER ACTION
   └─ User registers, deposits, invests, completes task, etc.

2. BACKEND PROCESSES
   └─ Route handler processes the activity

3. LISTENER TRIGGERED
   └─ ActivityListener.onActivityType() called with user data

4. PREFERENCES CHECKED
   └─ Query Firestore for user's email preferences

5. TEMPLATE SELECTED
   └─ Appropriate template selected for activity type

6. EMAIL GENERATED
   └─ Template rendered with user data

7. EMAIL SENT
   └─ Nodemailer sends via SMTP (Hostinger)

8. LOGGED
   └─ Email delivery logged to Firestore email_logs

9. USER RECEIVES
   └─ Professional branded email in user's inbox

(All automatic - no manual intervention needed!)

🔐 FIRESTORE COLLECTIONS
═══════════════════════════════════════════════════════════════════════════════

Collection: email_logs
  - userId: User's ID
  - email: Recipient email
  - emailType: Type of email sent
  - subject: Email subject
  - sentAt: Timestamp when sent
  - status: Delivery status (sent, failed, etc.)
  - messageId: SMTP message ID
  - Additional fields (amount, plan, taskTitle, etc.)

Collection: newsletter_subscribers
  - email: Email address
  - firstName: User's first name
  - lastName: User's last name
  - preferences: Email preference settings
  - subscribed: Is currently subscribed?
  - subscriptionDate: When subscribed
  - unsubscriptionDate: When unsubscribed (if applicable)

Collection: user_activities
  - userId: User's ID
  - activityType: Type of activity
  - activityData: Activity details
  - emailSent: Was email sent?
  - createdAt: Timestamp of activity
